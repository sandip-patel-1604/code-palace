Command

palace domains — Cluster Discovery

Automatically discover the domain structure of any codebase. No configuration, no manual tagging — just the import graph and git co-change patterns.

palace domains
$ palace domains

Domain Map (3 domains)
├── Auth  (3 files)
│   ├── src/auth/login.py
│   ├── src/auth/session.py
│   └── src/auth/token.py
├── Api  (3 files)
│   ├── src/api/routes.py
│   ├── src/api/handlers.py
│   └── src/api/middleware.py
└── Other  (2 files)
    ├── src/config.py
    └── src/main.py
Louvain community detection on the import graph. Co-change frequency from git history adds edge weight — files that always change together cluster together, even if they don't import each other directly.

Auto-naming picks the most common directory component. src/auth/* becomes Auth. No LLM needed — pure graph analysis.
Command

palace impact — Blast Radius

Before you touch a file, know exactly what breaks. Direct dependents, transitive reach, domain impact, git ownership, and a computed risk score.

palace impact src/core/models.py
$ palace impact src/core/models.py

╭────────────── Impact Analysis ──────────────╮
│  File:       core/models.py              │
│  Risk:       HIGH                        │
│                                           │
│  Direct dependents:     14              │
│  Transitive dependents: 47              │
│  Domain impact:   Auth (12), API (3) │
│  Primary owner:         alice (62%)     │
│  Churn (90d):           42 changes     │
│  Test files:            3               │
╰─────────────────────────────────────────╯
Risk scoring formula:
score = deps/5 + transitive/20 + churn/10 - tests

HIGH (≥5) — core model, wide blast radius, high churn
MEDIUM (≥2) — moderate impact, some coverage
LOW (<2) — leaf file, narrow scope

Supports symbol-level targeting:
palace impact models.py:User

palace search — Semantic Search

Find code by meaning, not just keywords. “authentication” finds login, validate_token, hash_password. Powered by ONNX embeddings — runs entirely offline.

palace search "user authentication login flow"
$ palace search "user authentication login flow"

Search: "user authentication login flow"  (6 results)

  Score  Kind      Name                    File
  0.89   class     AuthenticationService   src/auth/service.py
  0.84   method    login                   src/auth/service.py
  0.81   function  validate_token          src/auth/jwt.py
  0.79   class     User                    src/core/models.py
  0.76   method    get_user                src/api/users.py
  0.71   function  hash_password           src/auth/crypto.py
768-dimensional vectors from UniXcoder ONNX model. Each symbol is embedded from its name + signature + docstring. Query embedding is compared via cosine similarity in LanceDB. Filter by --kind class or --format json.
Command

palace diff — Change Impact

Understand the impact of any set of changes before you push. Domains affected, blast radius, risk level, and which tests to run.

palace diff HEAD~3..HEAD
$ palace diff HEAD~3..HEAD

Diff Impact: HEAD~3..HEAD (4 files changed)

  Changed Files:
    1. src/auth/session.py     [HIGH]
    2. src/api/middleware.py    [MEDIUM]
    3. src/config.py           [LOW]
    4. tests/test_auth.py      [LOW]

  Domains Affected:
    Auth [HIGH]  2 files modified
    Api  [MEDIUM]  1 file modified

  Blast Radius:       23 files transitively affected
  Overall Risk:       HIGH

  Suggested Tests:
    tests/test_auth.py
    tests/test_middleware.py
    tests/integration/test_login.py
Parses git diff --name-only, maps changed files to the palace index, runs impact analysis per file, and aggregates domain impact with the highest risk level. Works on any git range: HEAD~1..HEAD, main..feature, or a commit SHA.

Under the Hood

What Powers Phase 2

Four new infrastructure layers — all embedded, all offline, all zero-config.

▶
Git History
Subprocess git log --numstat parsing. Co-change analysis, ownership mapping, churn hotspot detection. No GitPython — zero extra dependencies for git analysis. Capped at --git-depth 10000 commits by default.
◆
Embeddings
ONNX Runtime + UniXcoder tokenizer. 768-dim vectors for every symbol. Cached at ~/.cache/code-palace/. One-time ~350MB download. XDG-compliant paths. --skip-embeddings to bypass.
●
Vector Store
LanceDB embedded. Zero-config, no server, stored in .palace/vectors/. Two collections: symbol embeddings and file embeddings. Cosine similarity search with optional kind and language filters.
■
Domain Clustering
NetworkX built-in louvain_communities(). Weighted graph: import edges (3.0) + co-change (up to 2.0) + semantic similarity (up to 1.5). Deterministic with seed=42. Auto-naming from directory prefixes.
