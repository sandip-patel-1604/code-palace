motivation

The Problem

"I got assigned a ticket on a codebase I don't know — where do I start?"

Every developer hits this wall. You clone the repo, open the file tree, and freeze. Which files do I touch? In what order? What breaks if I change this?

No existing tool answers this structurally — without an LLM, without reading all the docs, and without a senior engineer holding your hand. Code Palace is that tool: a graph-powered CLI that understands your codebase and tells you exactly where to start.

~/
No orientationNo map, no entry point guide, no dependency overview. Just a file tree and a ticket.
>_
No orderingWhich files to modify first? What are the leaves vs the roots in the dependency graph?
?!
No contextWhere does this symbol live? Who calls it? What does it import? You need a graph, not grep.

palace init — Zero-Config Indexing

Drop it in any project. No config files, no API keys, no Docker. It finds your source files, parses every symbol with tree-sitter, resolves import edges, and writes a queryable graph to .palace/.

Query dependencies or dependents. Traverse transitively. Output as tree, table, JSON, or DOT (for Graphviz).

palace plan — The Killer Feature

Describe a task in plain English. Palace extracts keywords, matches symbols, traverses the graph, and hands you an ordered file list — leaves first, roots last. No LLM. No hallucinations. Pure graph analysis.

palace plan — "add user authentication"

How It Works

Palace builds a three-layer graph from your source code. Each layer adds richer semantics, enabling increasingly powerful queries.

1
layer one
STRUCTURAL
The filesystem as a graph. Files, directories, modules, packages — and which language each belongs to.
files directories modules packages
2
layer two
SYMBOLIC
Every named entity parsed from source. Functions, classes, types, interfaces — with signatures, docstrings, and line ranges.
functions classes types enums interfaces
3
layer three
RELATIONAL
Edges between symbols and files. Imports resolve to targets. Calls link callers to callees. Inheritance chains connect types.
imports calls inherits uses

No infrastructure to spin up. No credentials to manage. No internet connection required. Just Python.

Language Support

Five parsers, all backed by tree-sitter grammars. Each extracts the symbol kinds idiomatic to that language.

Language Extensions Extracts
Python .py .pyi Functions, classes, methods, imports (absolute + relative), decorators
TypeScript .ts .tsx .js .jsx Functions, classes, interfaces, type aliases, enums, ES imports
Go .go Functions, methods, structs, interfaces, constants, grouped imports
Java .java Classes, interfaces, methods, enums, fields, package imports
C++ .cpp .cc .h .hpp Classes, structs, enums, functions, templates, namespaces, #includes
